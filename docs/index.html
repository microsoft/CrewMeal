  • OOXML 지오메트리를 근거로 삼아 환각 억제

HWP · HWPX → rhwp

포맷을 읽습니다

reads the format

닫힌 HWP를 네이티브로 파싱합니다. rhwp는 Rust+WebAssembly 기반 오픈소스 한글 엔진으로, LibreOffice 변환 없이 문단·표·수식·이미지 구조를 그대로 가져옵니다.

  • HWP 5.0(OLE2) · HWPX · HML 네이티브 파싱
  • 표·수식·도형 구조를 무손실로 보존
  • 오픈소스 · MIT · npm @rhwp/core

강화 전후, Copilot의 답변이 달라집니다

실제 고객 제안 PoC에서 캡처한 화면입니다. 표·도형·이미지로 표현된 정보는 강화 전에는 답하지 못하지만, 강화 후에는 근거·출처와 함께 답합니다. 이미지를 클릭하면 크게 볼 수 있습니다.

이미지 프로세스도 · PPT

통째로 이미지로 붙은 스윔레인

사업부·회계/자금팀·시스템·VAN/은행 레인의 외화 펌뱅킹 흐름도. 한 장의 이미지로 삽입돼 텍스트가 전혀 없습니다.

선택형 게시

환경에 맞는 검색 경로를 관리자가 선택

원본 파일 본문은 바꾸지 않습니다. 같은 의미 모델을 SharePoint 컬럼에는 Markdown으로, Copilot Connector에는 허용 태그 HTML로 게시합니다. 대상을 바꿀 때는 새 경로를 먼저 채우고 검증한 뒤 이전 경로만 정리합니다.

SharePoint 검색 컬럼

CrewmealSearchContent에 구조를 보존한 plain-text Markdown을 최대 63,999 UTF-16 단위로 저장합니다. 원본 ACL과 메타데이터를 그대로 사용하며, 최초 수동 재색인과 라이브러리 범위 Copilot 검증이 필요합니다.

Copilot Connector

허용 태그 HTML을 별도 externalItem으로 색인하고 원본 ACL을 복제합니다. 원본 라이브러리는 그대로 유지하면서 독립된 검색 스키마와 전체 콘텐츠를 사용합니다.

검증 후 안전 전환

관리자 포털에서 대상을 바꾸면 모든 문서를 새 경로로 다시 게시합니다. 검색 준비 상태를 확인하기 전에는 기존 게시물을 지우지 않으며, 실패한 작업만 재시도합니다.
