Microsoft · 오픈소스 PoC

Copilot을 위한 멀티포맷 콘텐츠를 더 맛있게 차려냅니다.

CrewMeal은 SharePoint의 PPTX·PDF·HWP 문서를 페이지 이미지로 렌더링하고 비전 LLM으로 분석해, 구조화된 검색 메타데이터로 강화합니다. 원본은 그대로 두고 Copilot 커넥터에 추가 색인만 얹어 검색 품질을 끌어올립니다.

핵심 기능

한 번의 파이프라인, 여러 포맷

모든 포맷은 동일한 SharePoint 인제스트 경로로 흐릅니다. 새 포맷을 켜면 인제스트·재조정·시연에 자동 편입됩니다.

🗂️

멀티포맷 문서 지원

PPTX·PDF·HWP를 페이지 이미지로 렌더링해 동일한 분석 스키마로 처리합니다. DOCX·XLSX는 구조가 준비되어 있습니다.

PPTX · PDF · HWP 구현
🔁

교체 가능한 분석 모델

프리미엄 비전 모델은 30페이지 발표자료 한 건에 토큰 비용만 1만 원이 넘습니다. 관리 포털에서 provider·배포·엔드포인트를 교체해 자사/오픈 호환 모델로 비용을 낮춥니다.

관리자 교체 UI
🔐

암호화 문서 복호화

MIP(Microsoft Purview) 및 기타 암호화 솔루션을 위한 복호화 파이프라인 훅. 관리자 토글로 on/off 하며, 활성화 후 실제 복호화 연동은 배포 환경에 맞춰 확장합니다.

구조 준비 · 토글 제공
📥

SharePoint 인제스트

SPFx 명령이 강화·삭제 요청을 큐에 적재하면 워커가 claim/lease로 자동 처리합니다. 원본 문서는 수정하지 않습니다.

opt-in 워크플로
🎛️

관리자 포털

대시보드·문서 상태·포맷 토글·모델 교체·복호화 설정·시연 업로드를 한 곳에서. 누적 토큰 비용도 실시간 추정합니다.

비용 가시성
💬

피드백 루프

상태 페이지의 튜닝 코멘트가 프롬프트에 주입되어 재작업되고, 피드백 코퍼스로 축적되어 JSONL로 내보낼 수 있습니다.

JSONL 내보내기
지원 포맷

포맷별 지원 현황

각 포맷은 관리 포털에서 개별로 켜고 끌 수 있습니다. 구현된 포맷은 기본 활성, 스켈레톤 포맷은 잠금 상태입니다.

포맷 확장자 처리 방식 상태
PowerPoint .pptx LibreOffice → PDF → 렌더 · 간트 geometry 근거 구현됨
PDF .pdf PyMuPDF 직접 렌더 · 페이지 텍스트 근거 구현됨
한글 (HWP) .hwp · .hwpx LibreOffice Writer 필터 → PDF → 렌더 구현됨
Word .docx 핸들러 등록 · 추출 로직 예정 구현 예정
Excel .xlsx 핸들러 등록 · 셀/표 추출 예정 구현 예정
실제 화면

구동 화면 캡처

아래는 실제로 앱을 구동해 캡처한 화면입니다. 이미지를 클릭하면 크게 볼 수 있습니다.

/admin/settings
설정: 포맷 토글 · 모델 교체 · 복호화포맷별 on/off, 비전 모델 provider/배포/엔드포인트 교체, MIP·기타 복호화 토글을 한 화면에서 관리합니다.
/admin
관리자 대시보드문서 상태 집계, 최근 작업, 누적 토큰 비용(추정)을 한눈에.
/admin/documents
문서 목록PPTX·PDF·HWP가 SharePoint·업로드 소스로 섞여 상태별로 나열됩니다.
/s/{token}
상태 페이지 (완료)단계별 타임라인, 소요 시간, 강화 비용(추정), 추출 HTML 미리보기를 사용자에게 제공합니다.
/s/{token}
상태 페이지 (진행 중)HWP 문서가 분석되는 동안 18/30쪽 같은 슬라이드 단위 진행률을 실시간으로 보여줍니다.
/s/{token}/html
강화된 구조화 HTML원본 페이지의 시각 콘텐츠가 제목·문단·표·목록의 허용 태그 HTML로 구조화됩니다.
/admin/tryout
시연 업로드문서를 직접 올려 동일 파이프라인을 시험합니다. 지원 형식이 활성 포맷에 따라 자동으로 표시됩니다.
파이프라인

문서 한 건이 강화되기까지

포맷 감지부터 색인까지, 모든 포맷이 같은 단계를 통과합니다.

01

감지 · 검증

확장자·매직·크기로 핸들러를 고릅니다.

02

복호화 훅

활성화 시 암호화 문서를 복호화합니다.

03

렌더링

페이지를 이미지로 변환합니다.

04

분석

비전 LLM이 strict JSON으로 분석.

05

HTML 생성

허용 태그 HTML로 렌더링.

06

색인

Copilot 커넥터에 externalItem 게시.
